IDEA + SpringBoot + maven 项目文件说明

.idea文件夹：存放项目的配置信息

包括数据源，类库，项目字符编码，历史记录，版本控制信息等。

（多为本地配置信息，进行版本控制时不用上传）

.mvn 文件夹：存放mvnw相关文件

存放着 maven-wrapper.properties 和相关 jar 包以及名为 MavenWrapperDownloader 的 java 文件

mvnw / mvnw.cmd 文件

全名是 MavenWrapper（Maven包装器），它的作用是在 maven-wrapper.properties 文件中记录你要使用的 Maven 版本，当用户执行 mvnw clean 命令时，如果发现当前用户的 Maven 版本和期望的版本不一致，那么就下载期望的版本，然后用期望的版本来执行 mvn 命令，比如 mvn clean 命令。

  • mvnw ：执行 mvnw 命令的 sh 入口（适配 Linux 系统）
  • mvnw.cmd ：执行 mvnw 命令的 cmd 入口（适配 Windows 系统）

src 文件夹

工作区文件夹

target 文件夹

gitignore 文件： 分布式版本控制系统 Git 的配置文件，意思为忽略提交！

在 .gitingore 文件中，需遵循相应的语法，即在每一行指定一个忽略规则。 如：.log、/target/、.idea

HELP.md 文件

帮助文档文件！

xxx.iml文件：Intellij idea 的工程配置文件

包含当前project的一些配置信息，如模块开发的相关信息，比如java组件，maven组件，插件组件等，还可能会存储一些模块路径信息，依赖信息以及一些别的信息。

pom.xml文件：项目对象模型（核心重要）

pom.xml主要描述了项目的maven坐标，依赖关系，开发者需要遵循的规则，缺陷管理系统，组织和licenses，以及其他所有的项目相关因素，是项目级别的配置文件。

IDEA中如何将代码块抽取成方法

快捷键：

  • 选中代码块之后，点击Ctrl+Alt+M

可视化界面：

  • 选中需要提取的方法，依次选择 Refactor –> Extract –> Method

自定义代码模板

  1. 打开Settings—Editor—Live Templates。

  2. 点击右侧加号(+),选择Templates Group，新增一个“模板”组，如custom

  3. 选中建好的custom，点击右侧加号(+)，新增一个Live Templates。

    • abbreviation：缩写，输入缩写快速生成代码。
    • Description ：模板描述。
    • Templates text ：模板文本，输出代码。

  4. 在模板文本中可以使用变量。

  1. 当模板创建完成之后，要选择模板的适用场景。
