File类

1 是什么

  1. File 类的一个实例对象，代表一个文件或一个文件目录。
  2. java.io.File
  3. File 类中涉及到关于文件或文件目录的创建、删除、重命名、修改时间、文件大小等方法，并未涉及到写入或读取文件内容的操作。如果需要读取或写入文件内容，必须使用IO流来完成。【总之就是文件和文件夹的除了读写之外的其它相关方法】
  4. File 类的对象常会作为参数传递到流的构造器中，指明读取或写入的”目的地”。
  5. 我们能利用 File 类实现的操作跟我们平时使用可视化界面操作文件/文件夹的操作大致上是相同的，只不过是使用代码实现而已。所以可以理解为：File 类就是利用代码操作文件/文件夹。

注意：

  • File 类没有无参构造方法(因为有重载的构造方法，JVM 就不再提供无参构造器了)。也就是说创建 File 实例必须指定文件/目录要保存的位置。
  • File 实例可以是一个文件或一个目录。
  • 创建File实例并不等价于创建了文件/目录。 创建 File 实例只是在内存中开辟了一块空间保存这个实例对象，并不会在硬盘上创建一个真实的文件或文件夹，只有进一步调用这个实例对象的 mkdir()、mkdirs() 或 createNewFile() 方法时才会在硬盘上创建出真正的文件/目录。

路径：

  • 相对路径：相较于某个路径下的路径。
  • 绝对路径：包含盘符在内的文件或文件目录的路径。

说明：
在 IDEA中：

  • 如果使用 JUnit 中的单元测试方法测试，相对路径即为当前 Module 下。
  • 如果使用 main() 测试，相对路径即为当前的Project下。

路径分隔符：

  • Windows 和 DOS 系统默认使用 \ 来表示；
  • UNIX 和 URL 使用 / 来表示。

mkdir() 、mkdirs() 和 createNewFile() 的区别

  • mkdir() / mkdirs() 方法用于创建目录，createNewFile() 方法用于创建文件。
  • 也就是说，在使用mkdir() / mkdirs() 方法时，即使开发人员指定名称为 aaa.txt ，它也不会创建出一个 aaa.txt 文件，而只是会创建出一个名为 aaa.txt 的目录。
  • mkdirs() 方法可用于一次性创建多级目录，而 mkdir() 方法每次只能创建单层目录。
  • 比如，要求的创建一个文件，这个文件保存在一个多层目录下，且这个多层目录中间的部分目录不存在，则创建文件和目录的方法都不会执行成功。不同的是 mkdir() 方法即使执行不成功也不会报错，而 createNewFile() 方法会直接报错。报错信息如下：

IO 流

是什么？

  • IO 流就是数据的流动。
  • 我们把数据的传输，看做是一种数据的流动，按照流动的方向，以内存为基准，分为输入流和输出流。
  • 即流向内存为输入流，流出内存为输出流。
  • 输入也叫做读取数据，输出也叫做写出数据。

IO的分类

  1. 按数据流向分类：输入流、输出流
  2. 按操作数据的单位分类：字节流、字符流

流的体系结构

常用结构

  1. 顶级父类：InputStream、OutputStream、Reader、Writer
  2. 文件流：FileInputStream、FileOutputStream、FileReader、FileWriter
  3. 缓冲流：BufferedInputStream、BufferedOutputStream、BufferedReader、BufferedWriter

  2. 创建相应的输入流，将 File 类的对象作为参数，传入流的构造器中

  3. 具体的读入过程；

  4. 关闭流资源。

  1. 创建 File 类对象，指明写出的数据的位置（不要求此文件一定要存在，如不存在在写出前会自动创建）；

  2. 创建相应的输出流，将 File 类的对象作为参数传入流的构造器中；

  3. 具体的写出过程；

  4. 关闭流资源。

注意

  1. 流的关闭原则：
  • 先开后关，后开先关。
  • 先关闭外层的流，再关闭内层的流。
  • 说明：关闭外层流的同时，内层流也会自动的进行关闭。

字节流

1 一切皆为字节流

  • 所有文件（文本、图片、视频等）底层都是以二进制形式保存在计算机中的，可以看做一个个的字节，传输时同样如此。
  • 所以，字节流可以传输任意文件数据。
  • 在操作流时，我们要时刻明确，无论使用什么样的流对象，底层传输的始终是二进制数据。

2 OutputStream

2.1、概述

  • java.io.OutputStream是抽象类，表示所有字节输出流类的超类。
  • 作用：将指定的字节信息写出到目的地。
  • 类中声明了所有字节输出流子类要实现的公共方法。

2.2 构造方法

  • OutputStream 类中没有重载构造方法，使用 JVM 提供的默认无参构造器！
  • OutputStream 是一个抽象类，不能直接 new 一个对象，但可以接收 new 出来的子类对象（多态）。

2.3 成员方法

ID 方法 返回值 说明
1 close() void 1.关闭输出流 2.释放系统资源
2 flush() void 刷新此输出流并强制任何缓冲的输出字节流被写出。
3 write(byte[] b) void 将 b 数组的内容写出到指定的文件中
4 write(byte[] b, int off, int len) void 将 b 数组中的指定内容写出到指定的文件中
5 write()[abstract] void 将该字节输出流写出到目的地。
  • 当完成流的操作时，必须调用close()方法，释放系统资源。

字符流

1 Reader

1.1 概述

  1. java.io.Reader是抽象类，是所有读取字符流类的超类。可以读取字符信息到内存中。
  2. 类中定义了所有字符输入流子类要实现的公共方法。
  3. 字符流只能操作文本文件，不能操作图片、视频等非文本文件。

字节流操作文本文件可能存在的问题及解决方案：

​ 当使用字节流读取文本文件时，可能会出现一个小问题：那就是遇到中文字符时，可能不会显示完整的字符。因为一个中文字符根据编码规则不同可能占用 2~3 个字节存储不等，而字节流读取是以单个字节为单位的，会出现一种情况就是：一个中文字符占了 3 个字节，使用字节流读取时会把这 3 个字节分开来展示，这样就会出现乱码。

​ 所以 Java 提供一些字符流类，以字符为单位读写数据，专门用于处理文本文件。

1.2 构造方法

  • Reader 类中没有重载构造方法，使用 JVM 提供的默认无参构造器！
  • Reader 是一个抽象类，不能直接 new 一个对象，但可以接收 new 出来的子类对象（多态）。

2 对象的序列化/反序列化机制

2.1 是什么？

  • 对象序列化机制允许把内存中的 Java 对象转换成平台无关的二进制流，从而把这种二进制流持久地保存在磁盘（文件）上，或通过网络将这种二进制流传输到另一个网络节点。
  • 当其它程序获取了这种二进制流，就可以恢复成原来的 Java 对象。【反序列化】

2.2 实现序列化的对象需要满足的条件

  1. 需要实现接口 Serializable；
  2. 当前类提供一个全局常量：serialVersionUID；
  3. 除了当前类需要实现Serializable接口之外，还必须保证其内部所有属性也必须是可序列化的。
    • 默认情况下，基本数据类型都是可序列化的。
    • 如果属性中包含了另外的类，则要求也必须是可序列化的。

补充：ObjectOutputStream 和 ObjectInputStream 不能序列化 static 和 transient 修饰的成员变量。

1 NIO

NIO；New IO；Non-Blocking IO

  • NIO 是从Java 1.4版本开始引入的一套新的 IO API，可以替代原来的 Java IO API。
  • NIO 与原来的 IO作用和目的相同，但是使用的方式完全不同，NIO 支持面向缓冲区的(IO是面向流的)、基于通道的IO操作。
  • NIO将以更加高效的方式进行文件的读写操作。
  • 随着 JDK 7 的发布，Java 对 NIO 进行了极大的扩展，增强了对文件处理和文件系统特性的支持，以至于我们称他们为 NIO 2.0。
