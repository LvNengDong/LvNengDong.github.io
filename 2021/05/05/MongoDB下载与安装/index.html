MongoDB下载与安装

Posted on 2021-05-05 In MongoDB

下载 & 安装

  • 使用宝塔的一键安装！

远端访问

在 Linux 上安装好 MongoDB 后，如果发现通过远程仍然不能访问到 MongoDB 数据库，这时主要需要检查两点：

  1. 是否放开了服务器的防火墙。

MongoDB 对外开放的端口号是 27017，必须放开此端口的防火墙。

  1. 设置允许所有地址访问。

MongoDB 默认只允许本机IP访问，如果需要其它IP能够访问，则需要修改配置文件。（下图是默认配置文件的配置信息）

  • 可直接设置配置文件为 bindIpAll: true，即允许所有的IPv4和IPv6地址访问

  • 也可以将 bindIp 修改为 0.0.0.0

  • 注意事项：
    net.bindIp 和 net.bindIpAll 是互斥的。也就是说，你可以指定其中一个，但不能同时指定两者。

MongoDB 数据库加密

MongoDB 的用户名和密码是基于特定数据库的，而不是基于整个系统的，所以所有的数据库都需要设置密码。

如果需要给 MongoDB 数据库使用安全验证，则需要用 --auth 开启安全性检查，只有认证通过的用户才能对数据库执行读写操作。

1、在服务器端进入 MongoDB 环境

1.1 MongoDB 安装在 Docker 中

如果 MongoDB 安装在 Docker 中，首先进入 Docker 的 MongoDB 环境中！

1.2 通过宝塔安装的 MongoDB

如果是通过宝塔安装的 MongoDB ，其默认的安装位置为：/www/server/mongodb （通过宝塔安装的所有应用都保存在 /www/server 目录下）

  1. 进入 MongoDB 的安装目录：
  1. 输入命令 mongo ，进入 MongoDB 环境

2、切换到 ‘admin’ 数据库

admin 数据库是 MongoDB 初始化时创建的，用于管理用户角色。

3、设置超级管理员用户

  • user：超级管理员的用户名
  • pwd：超级管理员的用户密码
  • roles：用于设置用户的权限，如：读、写等等。root 代表所有权限

验证是否添加成功：如果返回 “1” 表示添加成功，返回 “0” 表示添加失败。

4、给特定的数据库设置权限

第3步是给超级管理员设置账号和密码，现在要给特定的库设置权限。我这里以 beauty_dev 库为例。

4.2 添加用户

为 beauty_dev 库添加一个用户，并赋予权限。

  • 这段命令的意思是：创建一个普通用户，并给予读写权限，db 表示用户可操作的数据库名。

常见的权限类型：

Read：允许用户读取指定数据库

readWrite：允许用户读写指定数据库

dbAdmin：允许用户在指定数据库中执行管理函数，如索引创建、删除，查看统计或访问system.profile

userAdmin：允许用户向system.users集合写入，可以找指定数据库里创建、删除和管理用户

clusterAdmin：只在admin数据库中可用，赋予用户所有分片和复制集相关函数的管理权限。

readAnyDatabase：只在admin数据库中可用，赋予用户所有数据库的读权限

readWriteAnyDatabase：只在admin数据库中可用，赋予用户所有数据库的读写权限

userAdminAnyDatabase：只在admin数据库中可用，赋予用户所有数据库的userAdmin权限

dbAdminAnyDatabase：只在admin数据库中可用，赋予用户所有数据库的dbAdmin权限。

root：只在admin数据库中可用。超级账号，超级权限

5、重启

配置完成后，重启 MongoDB。

MongoDB 默认不开启访问控制，我们要通过 --auth 参数开启安全登录。--auth 一旦开启了，用户连接 MongoDB 的数据库时必须制定用户名和密码。
