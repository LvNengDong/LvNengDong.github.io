Posted on 2021-04-09 Edited on 2022-03-04 In Vue

Vue-cli

1、是什么？

通过对 WebPack 的了解，如果我们手动去设置打包相关的配置信息是非常麻烦的。因为在开发中，需要打包的东西不止是 Js、Css、HTML、image 等，还有更多的东西要处理，如果我如果我一一去添加这些插件和加载器非常耗费时间。

而作为一个开发人员，我们需要关注的往往只有项目源代码文件（即src文件）和少数几个配置文件。为了提高开发效率，Vue 官方提供了一个快速搭建 Vue Web项目的脚手架：vue-cli，使用它能快速的构建一个web工程模板。类似于 Java 中的 SpringBoot 等框架，我们只需在这个脚手架上进行二次开发即可！

3、模板项目的结构？

src

入口文件：main.js

单文件组件：*.vue*

我们称后缀名为.vue的文件为单文件组件。每一个.vue文件，就是一个独立的vue组件。而 App.vue 是整个项目的根组件。

单文件组件中包含三部分内容：

  • <template/>：Html 模板
  • <script/>：Js 脚本，这里编写的就是vue的组件对象，还可以有data(){}等
  • <style/>：Css 样式，支持CSS语法高亮和提示

每个组件都有自己独立的html、JS、CSS，互不干扰，真正做到可独立复用。

多文件组件

既然有单文件组件，那么就一定有多文件组件。多文件组件一般是一个文件夹，这个文件夹中除了包含一些单文件组件之外，还可以包含一些 image 等静态资源。所有这些资源形成一个大的组件对外暴露，我们把这个大的组件就叫做多文件组件！！！

.gitkeep

.gitkeep 是作用于 Git 进行版本管理的文件。

Git 进行版本控制时有一个特点就是如果某个文件夹为空就会自动忽略（即不对该文件夹进行版本控制），但是如果我们不想该文件夹被忽略的话，就可以在该文件夹下创建一个.gitkeep的文件（文件内容为空），这样Git在进行版本管理时就不会忽略该文件夹了！！！

4、运行

查看项目文件的配置信息：package.json

  • 可以看到这里引入了非常多的依赖，绝大多数都是开发期依赖，比如大量的加载器。
  • 运行时依赖只有 vue 和 vue-router
  • 脚本有三个：
    • dev：使用了webpack-dev-server命令，开发时热部署使用
    • start：使用了npm run dev命令，与上面的dev效果完全一样，当脚本名为“start”时，可以省略“run”。
    • build：等同于webpack的打包功能，会打包到dist目录下。
