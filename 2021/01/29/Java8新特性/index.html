Posted on 2021-01-29 Edited on 2022-08-21 In JavaSE , Java8新特性

函数式接口

包含且仅包含一个抽象方法的接口称为函数式接口。这句话不是规定接口中只能有一个方法，非抽象方法的个数是不加限制的。

当然在 JDK8 中接口中只能含有抽象方法和静态方法，所以这里的非抽象方法在 JDK8 中就是指静态方法，在 JDK9 中接口中还可以声明 private 方法。

@FunctionalInterface

我们通过在任一接口上使用 @FunctionalInterface 注解来声明当接口是一个函数式接口。如果某个接口是一个函数式接口，在 JavaDoc 中也会包含一条声明加以说明。

JDK内置四大函数式接口

接口类型 接口 抽象方法 说明
消费型 Consumer<T> void accept(T t) 对类型为T的对象进行操作。只进不出
供给型 Supplier<T> T get() 返回类型为T的对象。只出不进
函数型 Function<T R> R apply(T t) 对类型为T的对象进行操作，并返回类型为R的结果对象
断言型 Predicate<T> boolean test(T t) 确定类型为T的对象是否满足约束，并返回Boolean值

其它函数式接口

BiFunction<T, U, R>

  • 抽象方法：R apply(T t, U u)
  • 说明：拥有两个入参（Bi）的函数型接口

UnaryOperator<T> “一元运算符”

  • 抽象方法：T apply(T t)
  • 说明：UnaryOperator 是 Funtion 的子接口，所以其抽象方法是继承自 Function 的。UnaryOperator 是 Function 的一个特例，直译过来是“一元运算符”，与 Function 的区别就是“一元”（即只有一个未知数），UnaryOperator 的入参和返回值强制要求是同一种类型。

BinaryOperator<T> “二进制运算符”

  • 抽象方法：T apply(T t1, T t2)
  • 说明：BinaryOperator 是 BiFunction 的子接口，直接使用 BiFunction 的抽象方法，但要求两个入参和返回值均为同一类型。

BiConsumer<T, U>

  • 抽象方法：void accept(T t, U u)
  • 说明：一次可以消费两个入参的 Consumer

ToIntFunction<T> ToLongFunction<T> ToDoubleFunction<T>

  • 抽象方法：
    • int applyAsInt(T value)
    • long applyAsLong(T value)
    • double applyAsDouble(T value)

IntFunction<R> LongFunction<R> DoubleFunction<R>

  • 抽象方法：
    • R apply(int value)
    • R apply(long value)
    • R apply(double value)

Lambda表达式

Lambda 表达式就是对接口中抽象方法的具体实现代码。

Lambda 表达式解决了匿名内部类代码语法冗余的现象。

原来的匿名内部类 -->> Lambda 表达式

Lambda 表达式要求接口必须是函数式接口，即接口中只有一个抽象方法，所以在重写接口中的抽象方法时不会产生歧义，因此方法名就可以省略。

Lambda 是一个匿名函数，我们可以把 Lambda 表达式理解为一段可以传递的代码(将代码看成一个整体，像变量一样进行传递)。使用 Lambda 表达式可以写出更简洁、更灵活的代码。

Lambda 表达式整体可以视为一个看起来很长的变量，可以作为方法中的参数传递。

Lambda表达式，也可以称为闭包，是 Java8 的新特性。

Lambda 允许把函数作为一个方法的参数(函数作为参数进行传递)。

实现方式口诀：

拷贝小括号，写死右箭头，落地大括号。

  1. 拷贝小括号：将接口中抽象方法的参数列表全部拷贝过来；
  2. 写死右箭头；
  3. 落地大括号：具体的落地实现放在大括号中

函数

  • f(x) = kx + b，f(x) 是要求得的计算结果， kx+b 是计算过程。

Lambda表达式语法

格式

Lambda 表达式用于实现接口中的抽象方法，那么显然它应该包含 参数列表、方法体、返回值类型 三个部分。

Java8 中引入了一个新的操作符 ->，它将 Lambda 表达式分为两个部分：

  • 左侧：Lambda 表达式的参数列表；对应接口中抽象方法的参数列表；
  • 右侧：Lambda 表达式的方法实现体；对应接口中抽象方法的实现类的方法体。
  • 返回值类型根据方法体中的返回值自行推断。

Lambda表达式对接口的要求

  • 需要函数式接口的支持。即 @FunctionalInterface 接口

  • 因为函数式接口中有且仅有一个抽象方法，所以使用 Lambda 表达式实现的抽象方法是唯一确定的。

Lambda语法优化

  • 可选的类型声明：
    • 不需要声明参数类型，因为 JVM 编译器可通过上下文推断出数据类型(根据接口中的抽象方法推断)。
    • 可以不写参数类型，但是必须所有参数都不写参数类型。
  • 可选的参数圆括号：
    • 有且仅有一个入参时可以省略参数小括号；
    • 零个或多个入参都不能省略小括号。
  • 可选的大括号：
    • 如果方法体只包含了一条语句，可以省略大括号。
  • 可选的返回关键字：
    • 如果方法体中唯一的一条语句是 return 语句，可以省略 return 关键字和大括号。
    • return关键字和大括号要么全都省略，要么全都不省略。

方法引用

概念

在前面的学习中，Lambda 表达式中的方法的具体实现都是我们自己写的，但是究其本质，Lambda 表达式就是一个函数（方法），所以 Lambda 表达式中的方法实现并非一定要我们自己来实现，我们可以直接引用 Java 中现成的方法实现作为 Lambda 表达式的方法体，这就叫做方法引用。

可以理解为方法引用是 Lambda 表达式的另外一种表现形式。

语法格式

  • 对象::实例方法名
  • 类名::静态方法名
  • 类名::实例方法名

eg.1

Consumer<T> 是一个消费型接口，接口中的抽象方法是 void accept(T t)，一个参数，无返回值。

  • System.out 实际上指向了一个 PrintStream 类型的实例，所以采用 对象::实例方法名 的格式。

使用要求：

  • Consumer<T> 接口中的抽象方法 accept() 有一个参数无返回值，所以对应的 println() 方法也要是一个参数无返回值的方法。
  • 即：引用方法的参数列表和返回值类型 要和 函数式接口中抽象方法的参数列表和返回值的个数与类型 要一一对应，相当于用引用方法实现了接口中的抽象方法。

构造器引用

概念：

构造器引用就是将构造方法作为 Lambda 表达式的方法体。

我们需要声明一个函数式接口，将接口中的抽象方法作为对象生成器方法，调用时返回一个新创建的对象。

语法：类名::new

在调用接口中的抽象方法时，通过 Lambda 表达式的构造器引用(类名::new)来执行构造器方法创建对象。

注意：虽然构造方法无返回值，但是默认返回一个对象实例，所以其对应的函数式接口应该是有返回值的。

  • 存在的问题？

    Employee 的构造方法不只一个，当使用构造器引用时，具体调用的是哪一个构造方法呢？

  • 答案：

    引用的构造器方法必须和函数式接口中的抽象方法的参数列表一致。在本例中，Supplier<T> 接口中的抽象方法是 T get()，无参数，所以引用的构造方法必然是无参构造方法。如果使用 Function<T, R> 接口中的 R apply(T t) 抽象方法，那么引用的构造器就必须包含一个参数！

Tip:需要调用的构造器方法的参数列表要与函数式接口中抽象方法的参数列表保持一致！

数组引用

概念

数组引用就是把创建数组的方法封装到 Lambda 表达式的方法体中。

我们需要声明一个函数式接口，该接口作为数组对象的生成器，当调用接口中的抽象方法时，实际就会调用 Lambda 方法体中创建数组的方法创建一个数组并返回。

语法格式：Type::new

Stream API

Stream 是 Java8 中处理集合的关键抽象概念，它可以指定你希望对集合进行的操作，可以执行非常复杂的查找、过滤和映射数据等操作。简而言之，Stream API 提供了一种高效且易于使用的处理集合数据的方式。Stream 用于操作数据源（集合、数组等）所生成的元素序列。

注意事项：

  • Stream 自己不存储元素。
  • Stream 不会改变源对象。相反，它会返回一个持有结果的新的 Stream 对象。
  • Stream 操作是延迟执行的，这意味着链路上的所有操作都会等到需要结果的时候才开始执行。

Stream 操作的三个步骤

  1. 根据数据源（集合、数组等）创建一个 Stream 对象
  2. 中间操作：一个中间操作链，声明对第一步得到的流对象的处理方式
  3. 终止操作：开始执行中间操作链，并产生结果。

创建Stream对象

创建 Stream 对象的 4 种方法

  1. Java8 中的 Collection 接口被扩展，提供了两个获取流的方法(即所有的 Collection 子类对象都可以调用这两个方法获取 Stream 实例)。
  2. 通过 Arrays 类中的的静态方法 stream() 可以获取数组流。
  3. 通过 Stream 类中的静态方法 of() 获取流对象
  4. 通过 Stream 类中的静态方法 iterate 或 generate 创建无限流。
    • 无限流就是如果不加限制 ，将会一直生成数据的流。
类 方法 说明

Collection default Stream<E> parallelStream() 返回一个并行流
Arrays static <T> Stream<T> stream(T[] array) 返回一个流

多个中间操作可以连起来形成一个流水线，除非触发终止操作，否则中间操作不会对数据执行任何处理。只有在触发终止操作时一次性全部处理，称为“惰性求值”。

并行流与串行流

并行流就是把一个内容分成多个数据块，并用不同的线程分别处理每个数据块的流。

Java8 中将并行进行了优化，我们可以很容易的对数据进行并行操作。Stream API 可以声明性的通过parallel()与sequential()在并行流和串行流之间进行切换。

接口中的默认方法与静态方法

  • Java8 之前，接口中只允许定义抽象方法（只有方法的定义，而无方法的实现）；、
  • 从 Java8 开始，接口中可以存在方法的实现，但是这个方法必须被 default 关键字修饰。
  • 从 Java8 开始，接口中也可以定义静态方法。
  • 在接口中，静态方法（被 static 修饰的方法）和默认方法（被 default 修饰的方法）可以定义任意多个，不受数量的限制。

创建一个Optional对象

与 Guava 相似，Java8 中同样也没有提供创建 Optional 对象的公有方法，我们想要创建一个 Optional 对象必须通过 Optional 类提供的静态工厂方法。具体如下：

方法 说明
Optional<T> empty() 创建一个内部引用对象为null的非空Optional对象
Optional<T> of(T value) 创建一个Optional对象，要求参数中的 value 必须为非null
Optional<T> ofNullable(T value) 创建一个Optional对象，不对入参做特殊要求
（即可以为null，也可以非null）
